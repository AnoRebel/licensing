# Design Context — Licensing Admin

Scoped to `admin/` only. The rest of the monorepo (core libs, HTTP handlers,
Go ports) has no UI surface. This file is the single source of truth for
every visual decision in the admin console — every Phase-13 subtask reads
from here.

## Design Context

### Users

Backend and SRE engineers managing the license lifecycle for a deployed
licensing platform. Their context: they arrive at the admin only when
something needs doing — provision a new tenant, revoke a leaked key, rotate
a signing key after personnel change, investigate why a seat count is off.
Short focused sessions, high-stakes destructive actions mixed with
low-stakes lookups.

The job-to-be-done is **operational certainty**. They need to believe the
state they see is the state of the system, and that the action they click
is the action that fires. Nothing else matters.

### Brand Personality

Three words: **precise, typographic, unadorned.** The voice of the UI is
operational — verbs over adjectives, present tense, no marketing copy. It
should feel like Linear, Stripe's admin dashboard, and Vercel's project
dashboard — technical surfaces designed for people who already know why
they're here. Emotional goal: **confidence**, not delight. No whimsy, no
gamification, no empty-state illustrations with mascots.

### Aesthetic Direction

- **Dashboard-first, not marketing-first.** The skill's default baseline
  (`VARIANCE=8, MOTION=6, DENSITY=4`) is marketing-optimised and
  explicitly overridden here to **`VARIANCE=4, MOTION=3, DENSITY=8`** per
  the operational brief. Split-screen heros, asymmetric grids, and
  perpetual-motion bento cards are banned on admin pages.
- **Light + dark, both first-class.** Toggling mid-session must not cause
  FOUC. Wiring lives in `@nuxtjs/color-mode` with `classSuffix: ''`.
- **Typography carries the hierarchy.** Font-weight and muted-foreground
  colour do the work that scale and colour chips would do on a marketing
  page. Stack: **Cabinet Grotesk** (sans, Fontshare) + **JetBrains Mono**
  (numerics, Google) — self-hosted via `@nuxt/fonts`. Inter is banned.
- **Data renders monospaced.** Every number column (seat counts, expiry
  deltas, fingerprints, key-ids) uses `font-mono` so operators can
  compare rows vertically without saccade drift.

### References

**Yes:** Linear (typography, inline affordances, muted-chroma status
chips), Stripe Dashboard (data tables, destructive-action confirmations
with typed-to-confirm), Vercel dashboard (project list density, keyboard
navigation), 1Password Business admin (access management clarity).

**Not:** Any marketing landing page. Any dashboard with a 3-column
feature-card hero. Any tool that uses a rainbow gradient as a status
indicator. Any empty state with cartoon illustrations.

### Anti-references (explicit bans)

- No purple/blue "AI gradient" glow on primary actions
- No 3-column equal-card hero on the dashboard — use a 2-col zig-zag of
  metrics, or a bento grid with meaningful size differentiation
- No emoji anywhere (copy, toast messages, empty states, errors)
- No Inter as the primary font
- No `h-screen` anywhere — always `min-h-[100dvh]` or a grid-defined
  viewport
- No generic skeleton circles — loading states use inline row-shaped
  shimmer matching the actual layout being loaded
- No centered-text-over-dark-image hero anywhere

### Accessibility & Inclusion

- **Target: WCAG 2.2 AA** across every primary flow. Verified in CI by
  axe-core (task 13.12).
- **Contrast:** All foreground/background pairs derived from the oklch
  token set in `assets/css/tailwind.css` already satisfy AA at normal and
  large-text scales in both themes.
- **Motion:** Respect `prefers-reduced-motion`. All transitions wrapped
  so the `@media (prefers-reduced-motion: reduce)` path is transitionless.
- **Keyboard:** Every destructive action reachable by keyboard, focus
  ring always visible, modal traps keep `Tab` cycle contained.
- **Screen reader:** Status chips carry `aria-label` spelling out the
  state ("Active, expires in 12 days"). Icons are `aria-hidden` unless
  they're the sole label for a control, in which case they get an
  `sr-only` text companion.

### Design Principles

Five principles every Phase-13 subtask must honour:

1. **Legibility before flair.** If a visual flourish hurts scan speed on
   a 200-row table, it loses. Type hierarchy, tight vertical rhythm, and
   monospaced numerals beat any hover effect.
2. **Destructive actions demand friction.** Revoke, suspend, rotate, and
   delete require typed-to-confirm dialogs with the object's canonical
   identifier (kid, license id, scope name) echoed back. Never a single
   click, never a generic "Are you sure?".
3. **Latency has a visual grammar.** Optimistic UI only when the
   server-side op is idempotent and reversible. Every non-optimistic op
   shows a row-level pending state; the page never dims globally.
4. **State is always explicit.** A license is either active, grace,
   suspended, revoked, or expired — render the exact state word, never
   implicitly via colour alone. Colour is reinforcement, not semantics.
5. **Keyboard-first, mouse-second.** The five primary flows (sign in,
   create license, activate, revoke usage, rotate key) must each be
   completable without touching the mouse. Shortcut hints rendered in
   the UI where they exist.

### Token anchors (don't drift)

- Radius: `var(--radius)` = `0.625rem` — never hard-code `rounded-3xl`
- Accent colour: `--primary` / `--destructive` from the oklch palette —
  never introduce a second accent
- Shadow: cards use the token-driven `shadow-xs` scale from Tailwind v4;
  no custom `shadow-[...]` arbitrary values on admin surfaces
- Font scale: `text-xs`, `text-sm`, `text-base`, `text-lg`, `text-2xl`
  (dashboard metric) and `text-3xl` (page title, one per page)
- Grid gutter: `gap-6` (24px) between panels, `gap-2` (8px) inside
  form-groups, `gap-4` (16px) inside stat clusters

This file is **not** the component library — that's scaffolded via
`npx shadcn-vue add ...` when each subtask needs components. This file
gates every visual decision before it ships.
